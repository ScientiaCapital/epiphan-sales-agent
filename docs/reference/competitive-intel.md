## Blackmagic ATEM (Top Competitor)

### Positioning

**Their pitch:** Budget-friendly live switching ($295 - $2,795)

**Our counter:** ATEM is a switcher, not a recorder. Recording requires an external PC that can crash.

### When to Compete

- Recording is the primary need (not just switching)
- Multiple rooms require fleet management
- LMS integration is critical
- Mission-critical reliability matters
- Volunteer or non-technical operators
- 24/7 unattended operation required

### When to Walk Away

- Pure budget play under $1,000
- Customer heavily invested in ATEM ecosystem
- Heavy live graphics/virtual sets required
- Dedicated production staff comfortable with ATEM

### Key Differentiators

| Feature | ATEM | Pearl | Why It Matters |
|---------|------|-------|----------------|
| Recording | Requires external PC | All-in-one hardware | PC can crash |
| Fleet Management | None | Epiphan Cloud (200+ rooms) | Enterprise scale impossible with ATEM |
| LMS Integration | None | Panopto, Kaltura, YuJa, Opencast | Education market excluded |
| Reliability | Network drops, software freezes | Purpose-built, 24/7 capable | Cost of failed recording > price difference |

### Common Claims and Responses

| Claim | Response |
|-------|----------|
| "ATEM is cheaper upfront" | Calculate TCO: ATEM + PC + software + IT time = $3,000-5,000. Pearl Nano at $1,999 is all-in-one. |
| "ATEM has more I/O options" | For switching, yes. But Pearl records reliably. Do you need a switcher or a recorder? |
| "We've used ATEM for years" | For what use case? If recording is primary, Pearl is purpose-built for that. Keep ATEM for switching. |

### Proof Points

- NC State manages 300+ Pearl rooms vs. per-device ATEM management
- ATEM audio drift after 60 minutes makes long sessions "worthless"
- ATEM software freezes requiring hard reboot during live broadcasts
- Network connection limit of 6-8 simultaneous causes dropped connections

### Talk Track

**Opening:**
> "ATEM is a great switcher - probably the best value for live switching. But it's not a recorder. When recording is your primary need, you're bolting on a PC that can crash. Pearl is purpose-built for reliable recording."

**Closing:**
> "Keep ATEM for switching if you love it. But for recording - especially at scale - Pearl is the enterprise choice."

## vMix

### Positioning

**Their pitch:** Flexible Windows-based production software ($700 - $3,000+)

**Our counter:** Windows can crash. Hardware doesn't blue-screen during your most important event.

### When to Compete

- Reliability is paramount
- Non-technical operators (volunteers, staff)
- Multiple locations requiring standardization
- LMS integration critical
- 24/7 unattended operation

### When to Walk Away

- Dedicated IT staff comfortable with Windows
- Advanced customization required
- Already invested in vMix and loves it
- Budget under $1,000

### Key Differentiators

| Feature | vMix | Pearl | Why It Matters |
|---------|------|-------|----------------|
| Platform | Windows PC software | Purpose-built hardware | No Windows updates, driver conflicts |
| Reliability | Subject to Windows issues | No blue-screen risk | Can't fail on Sunday morning |
| Operator Skill | Requires Windows knowledge | 5-minute touchscreen training | Volunteers can operate |
| Long Duration | Memory leaks after 12+ hours | Built for 24/7 | Enterprise needs always-on |

### Common Claims and Responses

| Claim | Response |
|-------|----------|
| "vMix is more flexible" | Flexibility requires complexity. Pearl trades flexibility for reliability. |
| "vMix is cheaper" | Software is $700. Add reliable PC ($1,200+), capture cards ($300+), IT time - you're at $2,500+ with Windows risk. |
| "We've been using vMix for years" | How many times has Windows updated during an event? Pearl eliminates that risk. |

## Teradek

### Positioning

**Their pitch:** Emmy-winning wireless transmission ($1,500 - $16,000)

**Our counter:** Teradek streams but doesn't record well. SD card only, plus $600-3,600/year subscription.

### When to Compete

- Recording is primary need
- Fixed installation (not mobile)
- LMS integration required
- No tolerance for subscription fees
- Multiple inputs needed

### When to Walk Away

- Mobile wireless transmission is primary
- Film/sports production context
- Existing Teradek investment
- Single-source streaming only

### Key Differentiators

| Feature | Teradek | Pearl | Why It Matters |
|---------|---------|-------|----------------|
| Primary Function | Wireless transmission | Recording-first | Teradek doesn't record well |
| Recording | SD card only, single source | Multi-source to internal SSD | Enterprise needs more than SD cards |
| Subscription | Core.io: $49-299/month | No recurring fees | TCO: Teradek = $7,564+, Pearl = $3,999 once |
| LMS Integration | None | Native Panopto, Kaltura, YuJa | Education market excluded |

### Common Claims and Responses

| Claim | Response |
|-------|----------|
| "Teradek is used in Hollywood" | For wireless transmission. You need recording, not wireless. Different tools. |
| "Core.io is great for cloud management" | At $600-3,600/year. Epiphan Cloud is included. No subscription. |

## Quick Objection Responses

### Price Objections

| Objection | Response |
|-----------|----------|
| "Too expensive" | Calculate 5-year TCO with no recurring fees. |
| "Competitor is cheaper" | Add up total system cost including PC, software, IT time. |
| "No budget" | What's the cost of a failed recording during [important event]? |

### Switching Cost Objections

| Objection | Response |
|-----------|----------|
| "We're standardized on [X]" | Pearl integrates with [X], doesn't replace it. |
| "Locked into contract" | When does it expire? Let's plan for then. |
| "Team knows current system" | Pearl's touchscreen takes 5 minutes to learn. |

### "Current System Works" Objections

| Objection | Response |
|-----------|----------|
| "Current system works fine" | When it works. What's the cost when it doesn't? |
| "We've never had problems" | How long until a critical failure during [important event]? |
| "Don't fix what isn't broken" | Is remote management a pain? Fleet management solves that. |
